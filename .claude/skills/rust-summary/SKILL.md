# Rust 学習サマリー・復習問題生成スキル

`.claude/session/` 内の全ログを読み込み、学習内容を章立てで整理し、復習問題を生成する。

## 手順

### Step 1: ログファイルを収集する

```bash
find .claude/session -name "rust-learning-*.md" | sort
```

ファイルが存在しない場合は「ログファイルがまだありません。`/rust-log` を実行してセッションを記録してください。」と案内して終了する。

### Step 2: 全ファイルを読み込む

`find` で列挙した各ファイルを `Read` で読み込み、以下を抽出する:
- 学習したトピックの一覧
- 理解できたこと・定着した内容
- 繰り返しつまずいている点（複数ログで言及されているもの）
- 参照したドキュメントのリンク

### Step 3: Rust の主要カテゴリで章立てする

実際のログ内容に基づいて章を動的に構成する（空の章は省略）:

1. 所有権・借用・ライフタイム
2. 型システム・トレイト・ジェネリクス
3. エラーハンドリング（Result / Option）
4. コレクション・イテレータ
5. パターンマッチング
6. 並行処理・スマートポインタ
7. モジュールシステム・Cargo
8. テスト
9. その他（CLI, ファイルI/O など）

### Step 4: 難易度別の復習問題セットを生成する

「つまずいたこと」と「繰り返し登場する概念」を中心に問題を作成する:

- **基礎確認** (3〜5問) — 定義・用語・基本的な文法
- **コード読み取り** (3〜5問) — コードを見て動作を予測する
- **実装演習** (2〜3問) — 自分でコードを書く課題
- **応用・考察** (1〜2問) — 設計上の判断や比較を問う問題

問題はユーザーの実際の学習内容にパーソナライズする（汎用的な問題は避ける）。

## 出力形式

```markdown
# Rust 学習サマリー

**対象期間:** YYYY-MM-DD 〜 YYYY-MM-DD（ログ N 件）

---

## 学習チャプター

### Chapter 1: [タイトル]

**学習内容:**
- 要点

**定着した点:**
- 理解が確認できた事項

**要注意点:**
- 繰り返しつまずいた点や混同しやすい概念

**参考リンク:**
- [ドキュメントへのリンク]

---

## 復習問題セット

### 基礎確認

**Q1.** [問題文]
<details><summary>ヒント</summary>[ヒントまたは参照先]</details>

### コード読み取り

**Q1.** 以下のコードはどのように動作しますか？

```rust
// コードスニペット
```

<details><summary>答え</summary>[解説]</details>

### 実装演習

**Q1.** [実装課題の説明]

<details><summary>ヒント</summary>[ヒント]</details>

### 応用・考察

**Q1.** [考察・設計問題]

---

*生成日時: YYYY-MM-DD*
```

## 注意事項

- Learning モードのため、サマリーをファイルに保存する場合はユーザーに確認を取ること
- `<details>` タグで答えを非表示にし、自力で考えられる形式にする
