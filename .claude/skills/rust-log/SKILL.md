# Rust 学習ログ記録スキル

今日の Rust 学習セッションの内容を `.claude/session/rust-learning-YYYY-MM-DD.md` に保存する。

## 手順

### Step 1: 日付を取得する

```bash
date +%Y-%m-%d
```

### Step 2: セッションを振り返る

会話履歴全体を確認し、以下を整理する:

1. **学習したトピック** — 今日取り組んだ Rust の概念・機能
2. **理解できたこと** — しっかり理解・確認できた内容
3. **つまずいたこと** — 理解が難しかった点、エラーが起きた箇所、残っている疑問
4. **書いたコード** — 実装・修正したファイルやコードスニペット
5. **参照したドキュメント** — セッション中に参照したリンクや資料

### Step 3: 保存先ディレクトリを確認する

```bash
ls .claude/session/ 2>/dev/null || mkdir -p .claude/session
```

同日のファイル（`rust-learning-YYYY-MM-DD.md`）が既存の場合は、追記か上書きかをユーザーに確認する。

### Step 4: ファイル内容をユーザーに提示して承認を得る

**Learning モードのため、必ずファイル内容を先に提示し、「保存しますか？」と確認してから実行する。**

ファイルパス: `.claude/session/rust-learning-YYYY-MM-DD.md`

出力フォーマット:

```markdown
# Rust 学習ログ: YYYY-MM-DD

## 学習したトピック
- トピック1
- トピック2

## 理解できたこと
- 内容1
- 内容2

## つまずいたこと・疑問点
- 疑問・困った点1（→ 解決策や参照先があれば記載）
- 疑問・困った点2

## 書いたコード・修正したファイル
- `projects/xxx/src/main.rs` — 変更内容の概要

## 参照したドキュメント
- [リンクテキスト](URL)

---
*記録日時: YYYY-MM-DD HH:MM*
```

### Step 5: 承認後に保存し、完了を報告する

保存したファイルパスと内容の要約をユーザーに伝える。

## 注意事項

- `.claude/session/` ディレクトリは `.gitignore` により Git 管理外のため、個人的なメモも自由に記載してよい
- 同じ日に複数回実行した場合は、内容をマージして1つのファイルにまとめることを推奨する
