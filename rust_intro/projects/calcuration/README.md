# 四則演算プロジェクト
入力値に合わせて四則演算を行いその結果を標準出力する。

## 要件
数値と計算式が入力されたらそれに合わせて四則演算を行いその結果を標準出力します。

四則演算をサポートするため( `+`, `-`, `*`, `/`)の４つの計算モードを持ちます。


## 仕様
入力方法は直感性を重視し、通常の式と同様に入力します。

入力例
```
1 + 10
```

数値、演算式は、それぞれ1スペースで区切って入力される。

`()`や自乗など複雑な計算はサポート外とする。



### 入力
- データ型: i32 or char(1)
- i32は計算対象の値
- char(1)は演算式
- 入力数は計算の組になっている限り制限無し
- -1000 ~ 1000の入力値をサポート

### 出力
- データ型: i32
- 出力値: 1つの値
- 最小値: -1000000
- 最大値: 1000000

## テスト項目
- [x] 足し算のロジックで1 + 1 = 2になるか
- [x] 足し算のロジックで1 + -1 = 0になるか
- [x] 引き案のロジックで1 - 1 = 0になるか
- [x] 引き案のロジックで-1 - 1 = -2になるか
- [x] 掛け算のロジックで2 * 2 = 4になるか
- [x] 掛け算のロジックで-2 * 2 = -4になるか
- [x] 掛け算のロジックで-2 * -2 = 4になるか
- [x] 割り算のロジックで 4 / 2 = 2になるか
- [x] 割り算のロジックで -4 / 2 = -2になるか
- [x] 割り算のロジックで -4 / -2 = 2になるか
- [x] 割り算のロジックで 1 / 2 = 0になるか
- [x] 入力値が1 + 1が数値配列と式配列で`[1, 1]`と`[+]`に分解されているか
- [x] 入力値で `+`が有る場合にadd関数が実行されているか
- [x] 入力値が1 + 1の場合に2が返されるか
- [x] 入力値が1 + 1 + 1の場合に3が返されるか
- [ ] 入力値が1 ++ 1と不当な場合に正しくない入力ということ判別できるか
- [ ] 入力が 1 + 1 + と演算式で終わる場合も不当な入力と判別できるか
- [ ] 入力が a b c と不当な入力と判別できるか
- [x] 入力値が3つ以下の場合にエラーを返せているか
- [ ] 入力値に1000より大きい値が含まれる時にエラーが返されるか
- [ ] 入力値に-1000より小さい値が含まれる時にエラーが返されるか
